# En jeu \- Skribbl\.io

## `pickWord\(options\)`

Lets the current drawer choose one word among 3 options at the start of their turn\.

### Arguments

- **`options: string\[3\]`** — the 3 candidate words generated by the server\.
    - Constraint: must contain exactly 3 non\-empty strings\.

### Behavior

- Only visible&#x2F;usable by the **drawer** at the beginning of the round\.
- Starts a **selection timer \(10 seconds\)**\.
- When the drawer selects a word:
    - the client emits a Socket\.io event \(e\.g\., `select\_word`\) containing the chosen word,
    - the server stores it as the **secret word `W`** and generates a masked version **`W\_masked`** \(e\.g\., `\_ \_ \_ \_`\)\.
- After validation:
    - the game enters the **Drawing phase \(80 seconds\)**,
    - guessers see only `W\_masked` \(and the number of letters if your UI shows it\)\.

### Edge cases

- **Timeout \(10s\)** with no choice:
    - the server automatically selects one option \(e\.g\., random\) and starts the drawing phase\.
- If `options` is not exactly 3 items \(desync &#x2F; bug\):
    - client shows an error&#x2F;loading state and requests new options \(e\.g\., `request\_words`\)\.
- Non\-drawers must never be able to pick a word:
    - enforce both client\-side UI restriction and server\-side validation\.

## `drawOnCanvas\(action\)`

Allows the drawer to draw on the Fabric\.js canvas during the drawing phase\.

### Arguments

- **`action: CanvasAction`** — a drawing action captured from the canvas UI, typically containing:
    - coordinates &#x2F; points,
    - tool \(`brush` | `eraser`\),
    - brush size,
    - color,
    - optional: `actionId` or timestamp \(debug&#x2F;order\)\.

### Preconditions

- The caller must be the **current drawer**\.
- The game must be in **Drawing phase** \(round active\)\.

### Behavior

- Enables Fabric\.js editing mode for the drawer\.
- Applies each incoming `action` locally on the canvas:
    - brush draws strokes,
    - eraser removes pixels&#x2F;objects depending on your Fabric setup,
    - updates reflect instantly on screen\.
- Canvas tools exposed to the drawer \(depending on your UI\):
    - change color, change brush size, switch tool, undo, clear\.

### Postconditions

- The local canvas state is updated to include the new stroke&#x2F;object\.
- No scoring or word logic is handled here \(drawing only\)\.

### Edge cases

- If a non\-drawer tries to draw:
    - ignore the action and keep canvas read\-only\.
- If the drawing phase ends \(timer = 0 or round ended\):
    - disable drawing input immediately\.
- If `action` is malformed \(missing coords, invalid tool\):
    - ignore action and optionally log it\.

## `sendGuess\(guess\)`

Sends a guess from a guesser and triggers validation \+ scoring if correct\.

### Arguments

- **`guess: string`** — user input\.
    - Constraint: trimmed; length limited \(example: 1–80 chars\)\.

### Behavior

- Available only for **non\-drawers** during the drawing phase\.
- On submit:
    - client emits Socket\.io event \(e\.g\., `guess:send`\) with `guess`\.
- Server\-side:
    - normalizes input \(trim \+ case\-insensitive comparison\)\.
    - If `guess == W` \(correct\):
        - marks the player as **guessedCorrectly**,
        - awards guesser points using:<br>**`points = 100 × \(timeRemaining &#x2F; 80\)`**
        - awards drawer bonus: **\+50 per correct guess**
        - broadcasts updates \(`playerGuessed`, `scoresUpdated`, etc\.\)
    - If incorrect:
        - optionally treated as a normal chat message \(depending on your UI rules\), no points\.

### Edge cases

- Empty guess &#x2F; whitespace only:
    - blocked client\-side, rechecked server\-side\.
- Player already guessed correctly:
    - further guesses ignored for scoring \(no double points\)\.
- Drawer attempts to guess:
    - ignored&#x2F;rejected server\-side\.
- Chat security:
    - sanitize guesses before displaying \(prevent HTML&#x2F;script injection\)\.

## `generateCues\(timerState\)`

Reveals progressive letter hints during the round and updates the masked word display\.

### Arguments

- **`timerState: \{ timeRemaining: number, totalTime: number \}`**
    - For your game: `totalTime = 80`\.

### Behavior

- Runs during the drawing phase based on the synchronized timer\.
- Hint schedule:
    - at **60s remaining** → reveal 1 letter
    - at **40s remaining** → reveal 2nd letter
    - at **20s remaining** → reveal 3rd letter
- Updates `W\_masked` consistently for all players:
    - server is the source of truth, then broadcasts the new masked word\.

### Edge cases

- Very short words:
    - do not reveal more letters than the word length\.
- No duplicate reveals:
    - never “reveal” the same position twice\.
- Desync:
    - if the server corrects the timer, the client updates hint display immediately from server state\.

## `showLeaderboard\(scores\)`

Displays the real\-time leaderboard \(sorted by score\)\.

### Arguments

- **`scores: Array&lt;\{ playerId: string, username: string, score: number \}&gt;`**

### Behavior

- Displays players sorted by score descending\.
- Updates:
    - when a guess is correct \(guesser points \+ drawer bonus\),
    - and&#x2F;or at the end of the round \(final recap\)\.
- Read\-only UI: users do not interact with it\.

### Edge cases

- Ties:
    - keep stable ordering \(e\.g\., join order\) or define a tie\-break rule \(your choice, but be consistent\)\.
- Disconnected players:
    - either keep them visible until game end or remove them after a grace delay \(depending on your global rules\)\.

## `showWhoGuessed\(guessedList\)`

Shows which players have already guessed the word correctly\.

### Arguments

- **`guessedList: Array&lt;\{ playerId: string, username: string, timeRemaining: number \}&gt;`**

### Behavior

- Updates live as soon as the server validates a correct guess\.
- Can be displayed:
    - during the round \(“Alice, Bob…”\),
    - and&#x2F;or in the end\-of\-round summary\.

### Edge cases

- No duplicates:
    - one entry per player max\.
- Optional early round end:
    - if all guessers guessed correctly, server may end the round early \(only if you implement this rule\)\.

## `nextRound\(\)`

Closes the current round, resets round state, rotates the drawer, and starts the next word selection phase\.

### Arguments

- None \(usually triggered automatically by the server\)\.

### Behavior

Typical server sequence:

1. End the round \(timer reaches 0, or server decides to stop\)
2. Broadcast round recap:
    - reveal the word,
    - show who guessed,
    - freeze and broadcast scores
3. Choose the next drawer \(rotation\)
4. Reset round state:
    - clear canvas for everyone,
    - reset “guessedCorrectly” flags,
    - clear word state \(`W`, `W\_masked`\)
5. Start `pickWord\(\)` for the new drawer

### Edge cases

- Drawer disconnects:
    - server either skips to next drawer or applies reconnection&#x2F;grace logic\.
- Client desync:
    - on reconnect, server sends current round state \(drawer, masked word, timer, canvas state\)\.
